# PROMPT_FOR_DEVELOPMENT.md

Ты — senior frontend/fullstack разработчик и продуктовый инженер. Проект **уже в разработке** — local-first PWA RunBalance (Nuxt 4).

**Актуальный код и логика:** [docs/product/implementation-status.md](docs/product/implementation-status.md) и [AGENTS.md](AGENTS.md).

**Важно:** демо-моки в seed **не используются**. Все данные — от пользователя (онбординг, формы, завершённые тренировки) и `localStorage`.

## Идея продукта

Разрабатывается PWA-приложение для бега, объединяющее идеи:

- Strava: маршруты, история тренировок, спортивная мотивация;
- Nike Run Club: тренировки, подсказки, простой экран во время бега;
- recovery-приложения: готовность к нагрузке, восстановление, сон, усталость, HRV/пульс покоя;
- отдельная фича: ресурс кроссовок и автоматический учёт пробега обуви.

Приложение не должно быть глобальной социальной сетью в MVP. Основная цель — персональный беговой помощник.

## Главная продуктовая задача

Приложение должно отвечать пользователю на вопросы:

- что мне сегодня бежать;
- в какой пульсовой зоне бежать;
- где мне пробежать нужную дистанцию;
- стоит ли сегодня снижать нагрузку;
- как прошла тренировка;
- сколько ресурса осталось у моих кроссовок.

## Не входит в MVP

В первую версию не добавлять:

- глобальную социальную сеть;
- публичные сегменты;
- глобальные рейтинги;
- heatmaps;
- интеграции со всеми часами;
- полноценные аудио-тренировки;
- сложные клубы;
- глобальные челленджи;
- импорт FIT;
- админку модерации.

## Ключевые экраны

### 1. Главный экран «Сегодня»

Показывает:

- индекс готовности;
- рекомендацию на день;
- тренировку дня;
- целевую пульсовую зону;
- предложенный маршрут;
- выбранные кроссовки;
- кнопку «Начать тренировку»;
- объяснение, почему предложена именно такая нагрузка.

### 2. Экран выбора маршрута

Пользователь выбирает:

- стартовую точку;
- дистанцию: 3 км, 5 км, 10 км, 15 км или своё значение;
- тип маршрута: круговой, туда-обратно, свободный бег;
- предпочтения: больше парка, меньше дорог, минимум подъёмов.

Приложение предлагает маршрут и даёт пользователю подтвердить его.

### 3. Экран активной тренировки

Должен быть максимально читаемым на бегу.

Показывает:

- дистанцию;
- время;
- текущий темп;
- средний темп;
- пульс;
- текущую пульсовую зону;
- отклонение от целевой зоны;
- голосовую/визуальную подсказку;
- прогресс по тренировке;
- кнопки «Пауза» и «Завершить».

### 4. Экран результата

Показывает:

- дистанцию;
- время;
- средний темп;
- средний и максимальный пульс;
- время в пульсовых зонах;
- выполнение плана;
- карту маршрута;
- выбранные кроссовки;
- добавленный пробег кроссовок;
- субъективную оценку самочувствия.

### 5. Экран кроссовок

Показывает:

- список пар;
- пробег каждой пары;
- ресурс каждой пары;
- процент износа;
- статус: активные, скоро замена, архив;
- историю тренировок в выбранной паре.

## UI-направление

Использовать shadcn-style подход:

- карточки;
- аккуратные бейджи;
- крупная типографика;
- нейтральный фон;
- минимум визуального шума;
- bottom navigation;
- тёмный режим для активной тренировки;
- понятные статусы;
- mobile-first.

Для Vue/Nuxt использовать shadcn-vue или компоненты на базе Reka UI + Tailwind CSS.

## Технические требования

- Проект писать на TypeScript.
- Бизнес-логику не смешивать с Vue-компонентами.
- Компоненты должны быть преимущественно презентационными.
- Логику GPS, тренировки, восстановления, зон пульса, кроссовок вынести в composables/services/stores.
- Активная тренировка должна сохраняться локально, чтобы не потеряться при перезагрузке или потере сети.
- Все данные тренировки сначала сохранять локально, затем синхронизировать с backend.
- Recovery-рекомендации не должны выглядеть как медицинский диагноз.
- Интерфейс активной тренировки должен быть читаемым на бегу.
- Голосовые уведомления должны дублироваться визуальным баннером.

## Фактическая архитектура frontend

```text
app/
  stores/runBalance.store.ts    # единый Pinia store
  data/seedRunBalance.ts        # только empty* сущности
  services/                     # вся бизнес-логика
  composables/                  # GPS, голос
  components/ui/ + layout/
  pages/                        # все маршруты см. implementation-status.md
  plugins/splash.client.ts
app.html                        # HTML splash до Vue
public/icons/                   # PWA
```

Нет `features/` и отдельных domain-stores — не создавать без рефакторинг-задачи.

## Статус разработки (Phase 1–4)

Сделано: см. [implementation-status.md](docs/product/implementation-status.md).
