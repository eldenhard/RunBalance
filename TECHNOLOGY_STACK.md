# TECHNOLOGY_STACK.md

## Frontend

### Nuxt 4

Основной фреймворк приложения.

Почему:

- Vue-based стек;
- удобная структура проекта;
- SSR/SPA/SSG-гибкость;
- хорошая база для PWA;
- file-based routing;
- server routes при необходимости.

### Vue 3

Основной UI-фреймворк.

Почему:

- Composition API;
- удобная декомпозиция логики через composables;
- подходит под mobile-first PWA.

### TypeScript

Обязательный язык проекта.

Почему:

- доменная модель приложения достаточно сложная;
- нужны явные типы для тренировок, маршрутов, пульсовых зон, кроссовок и восстановления;
- проще поддерживать проект по мере роста.

### Pinia

State management.

Использовать для:

- профиля;
- тренировочного плана;
- активной тренировки;
- восстановления;
- маршрутов;
- кроссовок;
- истории тренировок.

### Tailwind CSS

Базовая стилизация.

Почему:

- хорошо сочетается с shadcn-подходом;
- удобно строить дизайн-систему;
- быстрый mobile-first UI.

### shadcn-vue / Reka UI

UI-компоненты.

Почему:

- shadcn-подход хорошо подходит для кастомной дизайн-системы;
- компоненты можно копировать и адаптировать;
- нет жёсткой зависимости от визуального стиля библиотеки;
- удобно делать карточки, кнопки, бейджи, табы, диалоги, формы.

### VueUse

Набор composables.

Потенциально использовать для:

- localStorage;
- online/offline state;
- device APIs;
- lifecycle helpers;
- sensors where applicable.

## PWA

### @vite-pwa/nuxt

PWA-интеграция.

Использовать для:

- service worker;
- web app manifest;
- install prompt;
- offline shell;
- кеширования статических ресурсов.

### IndexedDB

Локальное хранилище.

Использовать для:

- активной тренировки;
- GPS-точек;
- очереди синхронизации;
- черновиков тренировок;
- локальной истории до синхронизации.

### Service Worker

Использовать для:

- offline shell;
- кеширования;
- фоновой синхронизации там, где она поддерживается.

## Maps and Geo

### MapLibre GL JS

Карта в приложении.

Использовать для:

- отображения маршрутов;
- отображения GPS-трека;
- превью маршрутов;
- карты результата тренировки.

### GeoJSON

Формат маршрутов и треков.

Использовать для:

- сохранённых маршрутов;
- построенных маршрутов;
- трека тренировки.

### Browser Geolocation API

GPS-трекинг.

Использовать для:

- записи координат;
- расчёта дистанции;
- отображения текущего положения.

Ограничение: PWA может быть нестабильнее нативного приложения при работе в фоне и при заблокированном экране.

## Voice and Alerts

### Web Speech API / SpeechSynthesis

Голосовые уведомления.

Использовать для:

- превышения пульсовой зоны;
- отклонения от целевого темпа;
- смены этапа тренировки;
- уведомлений по дистанции/времени.

### Visual Alert Banner

Обязательный fallback.

Каждое голосовое уведомление должно иметь визуальный дубль.

### Vibration API

Опционально.

Можно использовать как дополнительный сигнал, если браузер и устройство поддерживают.

## Backend

### Вариант 1: Node.js + NestJS

Рекомендуемый вариант, если хочется единый TypeScript-стек.

Использовать для:

- REST/JSON API;
- авторизации;
- тренировок;
- маршрутов;
- профиля;
- кроссовок;
- истории;
- синхронизации.

### Вариант 2: Go

Альтернатива, если нужен более лёгкий backend и простая поставка бинарника.

Использовать для:

- API;
- обработки треков;
- геоопераций;
- фоновых задач.

## Database

### PostgreSQL

Основная база данных.

Хранить:

- пользователей;
- профили;
- тренировки;
- планы;
- кроссовки;
- recovery check-ins;
- настройки;
- историю.

### PostGIS

Геоданные.

Использовать для:

- маршрутов;
- треков;
- поиска маршрутов рядом;
- пространственных запросов;
- расчётов по геометрии.

### Redis

Опционально.

Использовать для:

- rate limiting;
- временных данных;
- очередей;
- кеша;
- фоновых задач.

## Storage

### S3-compatible storage

Опционально на будущее.

Использовать для:

- аватаров;
- превью маршрутов;
- экспортов;
- будущих медиа.

## Validation

### Zod или Valibot

Использовать для:

- схем форм;
- доменной валидации;
- API contracts;
- runtime validation.

## Testing

### Vitest

Unit-тесты.

Покрывать:

- пульсовые зоны;
- readiness score;
- shoe mileage;
- workout summary;
- voice alert rules.

### Playwright

E2E-тесты.

Проверять:

- онбординг;
- старт тренировки;
- завершение тренировки;
- выбор кроссовок;
- recovery check-in.

## Observability

### Sentry

Ошибки frontend/backend.

### OpenTelemetry

Трейсинг backend.

### Structured logs

Логи backend и критических frontend-событий.
