# Phase 1 Local-First PWA Implementation Plan

> **STATUS: COMPLETED** (2026-05-18). Mock data removed later — see [implementation-status.md](../../product/implementation-status.md).

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first runnable Nuxt 4 mobile-first PWA shell for RunBalance with shadcn-vue styling, mock product data, core pages, and a future-proof heart-rate source boundary.

**Architecture:** The first version is local-first frontend only: no backend, no auth, no real route building, and no live pulse on iOS/Safari. Domain types and mock data live under `app/types`, `app/data`, and `app/services`; UI pages consume those through small composables/stores so future IndexedDB, Capacitor BLE, and backend sync can replace mocks without rewriting screens.

**Tech Stack:** Nuxt 4, Vue 3, TypeScript, Pinia, Tailwind CSS, shadcn-vue/Reka UI, VueUse, @vite-pwa/nuxt, Vitest.

---

## File Structure

- `package.json`: project scripts and dependencies.
- `nuxt.config.ts`: Nuxt modules, PWA shell, CSS setup.
- `app.vue`: root app wrapper.
- `app/assets/css/tailwind.css`: Tailwind entry and design tokens.
- `app/components/layout/AppShell.vue`: mobile app frame and bottom navigation.
- `app/components/ui/*`: shadcn-vue primitives generated by CLI.
- `app/pages/index.vue`: Today page.
- `app/pages/plan.vue`: Plan page.
- `app/pages/start.vue`: Start workout page with mock route/workout setup.
- `app/pages/history.vue`: History page.
- `app/pages/profile.vue`: Profile page.
- `app/pages/shoes.vue`: Shoes page.
- `app/pages/workout/active.vue`: active workout UI with pulse unavailable/manual state.
- `app/pages/workout/result.vue`: result UI.
- `app/types/*.ts`: domain model types.
- `app/data/mockRunBalance.ts`: stable mock data for Phase 1.
- `app/services/heart-rate/heartRateSource.ts`: interface for `manual`, `mock`, future `capacitorPolarH10`.
- `app/services/shoes.ts`: shoe mileage/status helpers.
- `app/services/heartRateZones.ts`: zone helpers without live sensor dependency.
- `app/stores/*.ts`: Pinia stores for local UI state and mock entities.
- `tests/unit/*.test.ts`: focused tests for pure logic.

## Scope Notes

- No backend in Phase 1.
- No auth in Phase 1.

- No real route generation in Phase 1.
- ANT+ is future scope.
- Capacitor BLE wrapper is future scope, but `HeartRateSourceKind = 'capacitorPolarH10'` is reserved now.
